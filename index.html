『Webコンテンツコンテンツ作成』の成果

本テーマについて

HTMLや、CSS、JavaScriptを使ってWebページを作れるようにします。 そしてこれを1つだけを使ってページを作ったり、 CSSとHTMLや、HTMLとJavaScriptの2つを使ってページを作ったり、 もう少しレベルを上げると、 この3つとも全てを使ってWebページを完成できるようにようにします。

プロジェクト課題について

天気予報を調べられるWebページを作りました。 検索をかけると、都市名、緯度、経度、最高気温、天気、最低気温、湿度、風速、風向の 9個を表示させるプログラムです。また、表示はわかりやすいように単位をつけたり しています。Webページでも見やすいように、文字を中央に寄せたり、 工夫しています。色も可愛いページが良かったので、ピンクを基調に少し水色を入れて作りました。検索結果の文字は、 少し太くして、見やすくしてみました。

演習の成果: 世界の天気

数当てゲーム

課題３の内容は予想は４にしておいて答えをランダム表示にさせます。ボタンを設置させて、正解なら正解、 それ以降の入力では、すでにゲームは終わっていると表示させます。 そして不正解の場合では、その数よりもっと大きいか、もっと小さいかいか表示 させます。 4回目は、まちがい.残念でした答えは10です.と 表示させて、答えを強制的に表示させるプログラムです。 最後に結果の表示はコンソール内で出るようにします。 課題４の内容は、予想を決めていなく、 答えの数字の予想をプレーヤーにしてもらいます。 結果の表示方法は課題３と全く同じです。ですが課題３までは結果はコンソール上だけでの表示でしたが、 今回の課題４ではページ上に結果を表示させるようにます。

演習の成果: 数当てゲーム

各授業回の学習事項

第1回 WWW と HTML

今日は文字の書き方や、画像の出し方、見出しなどを習いました。 文字の書き方は<p>を使うとかけることを習いました。 また終了するときは</p>を使うことを習いました。 見出しは<h1>を使うと大きくなることを習いました。 そして数字を小さくしていくほど見出しが小さくなっていきます。 画像の出し方は、imgを使うとできることを習いました。

演習の成果: 演習1-4:のさまざまな要素

第2回 CSS による Web ページの装飾

今日はcssについて学びました。 cssとは、文字の色を変えられたり、背景の色、グラデーションなどの デザインを変えられたりします。 また、点線やドットなどもつけられたりします。 グリッドレイアウトでは、２×２（数字を変えれば違う数の表が作れる） の表も作れることを学びました。 そのおかげで、おしゃれなWebページを作れるようになりました。

演習の成果: ウェブブラウザの紹介

第3回 javaScript の基本

今日はJavaScriptについて学びました。 文字を表示させるときはconsoleを使うことを習いました。 また、コンソールやデバックを使い間違えを知れることも知りました。 ブレークポインについても学びました。 ブレークポイントは指定したとこで一時停止できることを知りました。

演習の成果: 演習3-6

第4回 DOM と javaScript

今日はボタンの表示のやり方、そしてその結果の 出し方を学びました。まずinputを使うとボタン表示できることを学びました。 また、type属性値もたくさんあることも知りました。 valueを使うことでユーザーが記入した値を知ることができるのを知りました。

演習の成果: 演習4-10

第5回 イベント処理

今日はJavaScriptからページ上に文字や画像、リンクの表示のやり方について学びました。 document.〇〇を基本的には使うことを学びました。 また、それらを活用して都市名を打つと都市番号をコンソール上に表示してくれるもの作るやり方まで、 学ぶことができました。

演習の成果: 演習5-7

工夫点，苦心点，強調点，感想

この授業を通して、私は本当にたくさんのことを学びました。 まず、HTMLについてです。私はこれまでHTMLを使ったことがなかったのですが、 この授業で初めて学ぶことができました。HTMLは、書いた内容がそのままウェブページ上に反映される仕組みになっており、 見た目の変化がすぐに確認できるので、 思っていたよりも難しくなく、わかりやすく、楽しく取り組むことができました。 書き方さえ覚えてしまえば、すぐに自分の思い通りのレイアウトを作ることができる点が印象的で良かったです。 また、ウェブページの開発に欠かせない「開発者ツール」についても知ることができました。 特に、エラーが出た時にその原因を探す際に使う「コンソール」機能はとても便利でしたが、 最初は少しわかりづらく、慣れるまで結構苦労しました。 そしてボタンの作成についても学びました。 HTMLでボタンを設置するところまでは簡単だったのですが、 そのボタンにJavaScriptで処理を加えて動作させる部分がとても難しく、 何度も繰り返し実行してすごく大変でした。 ホームページにリンクを貼る方法や、画像を挿入する方法も学びました。 これらの操作は、最初は不安でしたが、手順さえ理解すればそれほど難しくなく、 すぐできたので良かったです。 一番大変だったのは、やっぱりJavaScriptです。 普段のJavaとは異なり、JavaScriptはコンパイルしてからエラーを確認するのではなく、 ブラウザのコンソールでエラーを確認しなければならないため、最初はよく分からず戸惑いました。 さらに、ブラウザを更新してしまうと入力内容が消えてしまうことがあり、 何度もやり直す必要があるのが大変でした。 数当てゲームの課題も難しかったです。 正解を当てたときにプログラムを終了させる方法に苦戦しました。 しかし、解決してみると意外と簡単な方法で実現できることがわかり、 こんなに悩む必要なかったなと思いました。 特に工夫した点としては、CSSを使ってページのデザインにこだわったことです。 どうしても「かわいい」ページに仕上げたかったので、背景や文字の色合い、 ボタンのデザインなどを何度も調整して、 自分が納得いく「最高にかわいいピンク具合」に仕上げました。 CSSの使い方を覚えると、 自分の個性やセンスを表現できることがとても楽しかったです。 この授業を受けて、Webページの作り方だけでなく、その裏にある技術や考え方、 そして制作の楽しさや苦労も学ぶことができました。 実際に動くものを自分の手で作る喜びを知ることができたのは、 貴重な経験でした。将来の進路を考えるうえでも、 Webに関する分野を視野に入れてみたいと考えるようになりました。 今後もこの経験を活かして、より多くのことにチャレンジしていきたいと思います。
